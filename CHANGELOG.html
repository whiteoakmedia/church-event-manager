v1.6.0 May 9, 2026

Admin UX overhaul — Registrations, Reports, Check-In

Comprehensive polish pass on the admin pages your volunteers and staff use most. The Registrations table, Reports dashboard, Email Center, and Check-In screen all got meaningful upgrades.

Registrations admin

  • Filter form and bulk-actions toolbar are now one combined toolbar — visually and mechanically a single unit.
  • New date-range filter (registered between X and Y) plus a quick "Checked in / Not checked in" filter.
  • New "Type" column shows the registration tier (e.g. Adult / VIP) inline on every row, batch-loaded so big tables stay fast.
  • Live "N selected" counter next to the bulk-action dropdown.
  • Replaced hand-rolled pagination with WordPress-native paginate_links() — scales cleanly past 20+ pages.
  • The CSV export button now preserves your active filters: what you see is what gets exported.

CSV export

  • Added first-class Registration Type and Type Price columns so financial reporting works without extra spreadsheet wrangling.
  • Internal meta keys no longer surface as junk Custom:* columns.
  • Filename uses your church's local timezone, not the server's.

Reports dashboard

  • Added a date-range filter with one-click 7-day / 30-day / 90-day presets.
  • Replaced the homemade bar chart with a proper Chart.js line chart. Days with zero registrations now render as a continuous flat line instead of breaking the timeline.
  • New doughnut chart breaking down registrations by event category.

Email Center

  • Added two real-world recipient filters: Checked-in attendees (post-event thank-yous) and No-shows (registered but didn't come).

Check-In page

  • New "Add Walk-in" button + modal — type a name, tap submit, the person is registered and checked in in one step. Email is optional. The list refreshes automatically.

Polish

  • Replaced jarring emoji throughout the admin (drag handles, remove buttons, status messages) with proper WordPress dashicons.
  • Registration delete confirmation now names the person being deleted and explains exactly what gets removed.
v1.5.0 May 9, 2026

Comprehensive bug-fix pass after audit

The most important release in this window. Closed a privacy hole, fixed a race condition that could oversell capacity, repaired the calendar's end-of-month and bi-weekly logic, and stopped a class of silent admin errors.

Privacy

  • Closed a registration-disclosure bug. The "look up my registrations" page used to render results inline whenever anyone typed in an email — meaning anyone who knew your address could see what you'd signed up for. The form now always returns the same generic confirmation regardless of whether the email matches anything, and dispatches the list as an email instead.
  • Added CSRF protection to the lookup form.

Data integrity

  • Fixed a waitlist race condition. Two near-simultaneous cancellations could each promote a different waitlister into the same single open spot. Promotion now runs inside a database transaction so this is impossible.
  • Eliminated a duplicate-email bug where waitlist promotions sent both a confirmation email and a "you're off the waitlist" email — now only the latter goes out.
  • Closed a capacity-bypass loophole. A registrant could sign up with 1 attendee then edit their party size to 50 without revalidation. Capacity, category caps, and per-event maximums are all re-checked on edit.
  • Deleting a registration now properly cleans up associated check-in, waitlist, and meta records instead of leaving orphans behind.

Calendar

  • Last-day-of-month evening events no longer disappear. The query was filtering by date instead of date+time, so anything after midnight on the upper boundary was silently excluded.
  • Events with missing start times no longer get dumped onto the 1st of the month.
  • Bi-weekly groups now use a stable schedule instead of a calculation that broke between calendar years.
  • Monthly groups now actually behave monthly (they were silently rendering 4× per month).

Admin reliability

  • The "Notify Email" field can no longer be silently wiped by a typo.
  • Garbage in the event date/time field can no longer silently move an event to January 1, 1970.
  • Custom-field waiver text now preserves line breaks (was being mashed into one paragraph).

Notifications

  • Activated the dormant capacity-alert system. The function existed but was never actually called. You now get an email once when an event hits 80% full and once when it hits 100%, throttled so it doesn't spam you on every subsequent registration.
v1.4.2 May 9, 2026

Events with no end time were silently disappearing

If an event was created without an end time, it was being filtered out of the public listing entirely. Eight upcoming events were showing as six. Fixed.

v1.4.1 May 9, 2026

Multi-recipient admin notifications, restored

When you put a comma-separated list in the "Notify Email" field, the plugin was mashing all the addresses into one invalid combined email. Notifications were going to a non-existent address.

  • Save logic now correctly splits the list and validates each address.
  • Send logic delivers a separate email to every recipient.
  • Existing mangled values are auto-repaired on plugin upgrade.
v1.4.0 May 9, 2026

Behind-the-scenes error reporting

Whenever the plugin encounters a problem on your site, it now phones home to White Oak Media's portal automatically. We see it, we fix it — without you needing to file a ticket.

  • No registrant data is sent. Only error type, file, line number, and plugin version.
  • Throttled to once per hour per unique error so we never get flooded.
  • Disabled on shared environments by setting one toggle if you ever want to.
v1.3.4 May 9, 2026

Registration redirect & resilience

Made the post-registration redirect a fixed destination (the Church Events page) and wrapped the email-notification step in a safety net so a notification failure can never break a registration.

v1.3.3 May 9, 2026

"After Registration Redirect" field returned

Restored the per-event redirect URL field in the event editor and added CSS to suppress theme-injected publish timestamps on group pages.

v1.3.0 — 1.3.2 May 9, 2026

Group page visual upgrades

Group cards and single-group pages without a featured image used to look bare. They now render a polished gradient hero based on the group type — Bible Study, Men's, Women's, Youth, Outreach, etc. — each with its own color treatment. Removed the random initial-letter placeholder per your feedback.

v1.2.8 — 1.2.9 Mar 2026

Foundational stability fixes

Resolved the original "internal server error after registration" issue, fixed events incorrectly displaying as "Past" in the admin, and added the /events → /church-events/ redirect.

  • Registration AJAX endpoint hardened against PHP 8 fatals.
  • Status column now compares against your church's local time, not server UTC, and respects the event's end time.
  • Old /events URL now 301-redirects to the canonical events page.
